⚠ Destroy ZFS Pool

This will permanently destroy the pool and all datasets within it. The underlying disks will not be wiped, but all ZFS data will be lost.

This action cannot be undone.

⚠ Force Import Pool

ZFS reports this pool was previously in use by another system or was not cleanly exported.

Only force-import if you are certain this pool is no longer in use elsewhere. Forcing import on an active pool can cause data corruption.

⚠ Dataset Has Children

has child datasets and cannot be deleted normally.

Recursive delete will permanently destroy this dataset and all of its children, including their data. This cannot be undone.

⬆ Upgrade ZFS Pool

This will upgrade your pool to support all available ZFS feature flags on this system.
